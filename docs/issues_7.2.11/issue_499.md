First of all, thank you for your good work. Your app is great!
Since I have upgraded to version 7.2.11, I have been experiencing 3 issues:
1. The resuming download feature for videos is broken
2. CPU usage is extremely high
3. Firefox add-on in permanent private mode does not save the privacy selection and it pop-ups the request every time I launch the browser.
I would also like to request a feature, if possible: 4) Please just add the options 3 and 4 in the drop-down list of the numbers of simultaneous downloads. Why 1, 2, and then skip to 5? The first four numbers are the most useful for most sites, in my opinion. In the future, if you will have more time, perhaps you could decide to allow users to input the desired number directly. Thanks!
ISSUE 1 If I wanted to update the source URL of a paused video download, in the past I could simply click on "refresh link" and then select the new URL by clicking on one of the offered downloads in the floating "DOWNLOAD VIDEO" widget. This feature is broken since I updated to 7.2.11. If I repeat the same procedure, nothing happens.
ISSUE 2 I am temporarily using a low end laptop (AMD Ryzen 3 3250U, 8 GB ram, SSD) and 2 simultaneous download with 4 threads each are enough to get a 100% CPU load. System is very clean, with the bare minimum number of applications running in the background.
ISSUE 3 I use Firefox in permanent private mode (Options > Privacy > History > Firefox will never remember history). Despite that, all my other add-ons do not lose their settings when I close the application. Since the introduction of the privacy prompt for the XDM add-on, every time I open the browser I have to select the choice again. Disabling permanent private mode solves the issue, but that is of course not an option for me.
- OS: Windows 10
- Browser: Firefox
- XDM addon Version: 2.2
- XDM Version: 7.2.11
