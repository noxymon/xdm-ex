# Settings/Change Download Location Very Slow with Versions 7.210 & 7.2.11 (#485)

[Settings/Change Download Location Very Slow with Versions 7.210 & 7.2.11](https://github.com/subhra74/xdm/issues/485#top)

[schmidtp72](https://github.com/schmidtp72)
[on Feb 9, 2021](https://github.com/subhra74/xdm/issues/485#issue-804197197)
I've been using XDM for a while and just installed the latest version (7.2.11) on a new desktop computer.
When I tried to setup different paths under Tools/Options and click on the change button, the program freezes. Eventually (after a few minutes) the dialogue box will open up to change locations. However even that's very slow. The default location is set to C:\user (which is an SSD). I've changed it to another installed HDD J:\XDM I downloaded the two previous versions. The same fault appears in 7.2.10. However 7.2.08 works fine.
So to reproduce:
1. Click on Tools
2. Click on Options
3. Scroll down to Default download folder
4. Click on Change button
5. The program becomes unresponsive for a minute or so. Then finally opens up a dialogue box
6. Using anything in the dialogue box is very slow
The same fault happens when you click on Change Temporary Folder as well.
I ended up just changing the default directories with V7.2.08 and reinstalled the latest version over the top.
- OS: Windows 10
- Browser: Chrome and Firefox
- XDM addon Version - 2.2
- XDM Version 7.2.11 and V7.2.10
